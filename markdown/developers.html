# Contributors documentation This document summarizes information for DDF committers and contributors. It includes information about the development processes, policies and development guide. --- * <a href="#welcome">Welcome!</a> * <a href="#workflow-and-policies">Workflows</a> * <a href="#report-bug">Report bug</a> * <a href="#contribute">Contribute</a> * <a href="#pull-requests">Pull requests</a> * <a href="#request-feature">Request a new feature</a> * <a href="#best-practices">Best practices</a> * <a href="#best-practices-testing">Testing</a> * <a href="#best-practices-development">Development</a> * <a href="#questions">Questions?</a> --- <a name="welcome"></a> # Welcome! Welcome to DDF project, if you're reading this document then you're interested in contributing to DDF, thanks for that. DDF is exciting project and we need lot of contributions in documentation, feedbacks, ideas, bug reports etc. We deeply want you to be one of the DDF's owners. # Workflows This section explains all basic necessary activities in contributing to DDF. <a name="report-bug"></a> ## Report a bug To issue a bug you need to create an issue on <a href="https://github.com/ddf-project/DDF/issues" target="_blank">on github</a>. To help us solve issues eficiently, you can provide us with following information: 1. Include error message and log, we recommend to use gist to paste log. 2. Short description how to reproduce the bug. <a name="contribute"></a> ## Contribute If you want to contribute code, for the first ten developers all request are approved. After that we will have a light-weight approval process for your request. You can register <a href="http://adatao.com/forms/ddf.html" target="_blank">here</a> and let us know more about your thinking. If you want to contribute to other areas, please let us know more [here](http://adatao.com/forms/ddf.html). We welcome all contributors. <a name="pull-requests"></a> ## Pull Request ### Create a Pull Request You can create pull request on DDF project. Please follow instruction <a href="https://help.github.com/articles/creating-a-pull-request" target="_blank">on github</a>). In addition to that we recommend you to give detail description on the Pull Request about the problem statement, goals and solutions of pull requests. ### Approve Pull Request At this phase of the project, all Pull Request will be approved by admins of the project. They will take a look, test it and response within 72 hours. In general all PR need to qualifed thw following criteria: 1.Pass all unit test. 2.Code format is correct. <a name="request-feature"></a> ## Request a new feature We welcome all ideas and comments. Before you request feature, please take a look at our current <a href="plans.html" target="_blank">roadmap</a>. To request a new feature you should open an issue in our <a href="https://github.com/ddf-project/DDF/issues" target="_blank">issue tracker</a> and summarize the desired functionality. You can also send a message to <a href="https://groups.google.com/forum/#!forum/ddf-project" target="_blank"> DDF google group</a>. <a name="best-practices"></a> ## Best Practices <a name="best-practices-testing"></a> ### Testing We use maven to run unit test. You can also use sbt test. All pull request needed to pass the whole test suites. <a name="best-practices-development"></a> ### Development We use Eclipse and IntelliJ IDE to write code. It's required to import our java-formatter into your IDE. <a name="questions"></a> ### Questions? If you have any questions after reading this document, then please reach out to us via the <a href="https://groups.google.com/forum/#!forum/ddf-project" target="_blank">DDF's google group</a>
